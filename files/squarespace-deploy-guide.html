🌿 bighatlawn.com — Squarespace Deployment

Step-by-step guide to deploy all 512AI tools on Rudy's Squarespace site

⏱ ~45 min total to deploy everything
0 / 0 steps done
1
Chat Widget (~5 min)
No Squarespace plan upgrade needed
1
Log into Squarespace as bighatlawn.com admin
account.squarespace.com → Log In → bighatlawn.com
Note: You need Editor or Admin access. If Rudy gives you Editor invite, accept it first from your email.
2
Navigate to Code Injection (Settings → Advanced)
Squarespace Dashboard → Settings → Advanced → Code Injection
Squarespace plan note: Code Injection requires the Business plan or higher. If you don't see "Advanced" in Settings, Rudy needs to upgrade from Personal → Business (~$23/mo).
📸 What you should see: Settings panel → left sidebar → "Advanced" → Code Injection page with "Header" and "Footer" text areas
3
Paste chat widget script into Footer injection
Settings → Advanced → Code Injection → Footer text area

Paste this exactly as-is. The defer attribute means it won't slow down page load.

<!-- 512AI Big Hat Lawn Chat Widget --> <script src="https://cdn.jsdelivr.net/gh/kalelra/512ai@main/files/widget-bighatlawn.js" defer></script>
Put this in Footer (not Header). Footer injection runs after the page loads — better for performance.
Click Save. Preview the site — green bubble should appear bottom-right within 2 seconds.
4
Verify chat widget is live
  1. Open bighatlawn.com in a new incognito tab
  2. Wait 2 seconds — green bubble should appear bottom-right
  3. Click it — chat panel should open with greeting
  4. Type: "How much does mowing cost?"
  5. Verify AI responds with pricing info
AI responds correctly and the session ID shows in chat header
2
Voice AI Widget (~5 min — after Bland.ai key is added)
Deploy same way as chat
5
Add BLAND_API_KEY_BHL to Railway backend
railway.app → 512ai-backend project → Variables → Add variable
Railway auto-redeploys when you add variables. Wait ~60 seconds for the new deployment before testing.
6
Paste voice widget script into Footer injection
Settings → Advanced → Code Injection → Footer

Add this below the existing chat widget script tag — don't replace it.

<!-- 512AI Big Hat Lawn Voice Widget --> <script src="https://cdn.jsdelivr.net/gh/kalelra/512ai@main/files/voice-bighatlawn.js" defer></script>
The voice button appears at bottom: 92px right: 24px — directly above the chat bubble. Both will coexist.
Save → Preview — phone icon should appear above the chat bubble
3
Schema Markup + SEO (~15 min)
Biggest impact for Google ranking
7
Add LocalBusiness schema to Header injection
Settings → Advanced → Code Injection → Header text area

This goes in Header (not Footer). Open seo-bighatlawn.html and copy the full schema block — fill in Rudy's real phone, address, and social links first.

Fill in all 🔴 placeholders in the schema before pasting. Invalid placeholders will be read by Google and look bad.
After saving, test at: search.google.com/test/rich-results → should show LandscapingBusiness
8
Set Homepage page title and meta description
Pages → Home (⚙️ gear icon) → SEO tab
Page Title: Big Hat Lawn | Austin Lawn Care & Mowing Service Meta Description: Big Hat Lawn offers professional lawn mowing, cleanup, and maintenance in Austin, TX. Get a free quote — AI scheduling available 24/7.
Browser tab should show new title when visiting bighatlawn.com
9
Add business phone + address to footer
Pages → scroll to Footer → Edit → add text block

Add Rudy's NAP (Name + Address + Phone) consistently here. Example:

Big Hat Lawn Austin, TX | (512) 555-0100 | info@bighatlawn.com Licensed & Insured | Serving all of Austin and surrounding areas
Google uses NAP consistency across the web for local ranking. This + GBP + Yelp all having the same info = better ranking.
10
Connect Google Search Console
Squarespace Settings → Marketing → Google Search Console → Connect
  1. Click "Connect" → sign in with Rudy's Google account
  2. Squarespace auto-verifies ownership
  3. Go to search.google.com/search-console
  4. Sitemaps → Submit → enter bighatlawn.com/sitemap.xml
GSC shows bighatlawn.com as a verified property
4
LawnPro CRM Webhook (~10 min — after CRM access)
Activates scheduling sync + SMS automation
11
Deploy lawnpro webhook function to Netlify (512ai.co)

Copy the webhook file to the Netlify functions folder and push to GitHub (auto-deploys).

cd ~/Claude_Projects/512ai cp files/lawnpro-webhook-bighatlawn.js netlify/functions/lawnpro-webhook-bighatlawn.js git add netlify/functions/lawnpro-webhook-bighatlawn.js git commit -m "feat: add LawnPro webhook handler for Big Hat Lawn" git push origin main
After pushing, Netlify auto-deploys. Function available at:
https://512ai.co/.netlify/functions/lawnpro-webhook-bighatlawn
12
Register webhook in LawnPro
LawnPro Admin → Settings → Integrations → Webhooks → Add New
Webhook URL: https://512ai.co/.netlify/functions/lawnpro-webhook-bighatlawn Events to subscribe: ✅ job.created ✅ job.updated ✅ job.completed ✅ job.cancelled
Copy the webhook secret that LawnPro shows after saving. Add it to Railway + Netlify as LAWNPRO_WEBHOOK_SECRET.
LawnPro shows webhook as Active. Test with a sample job — check function logs in Netlify dashboard.
5
Final Verification (~5 min)
13
Run the pre-launch QA checklist

Open qa-launch-checklist.html and run every test. All critical items must pass before announcing to customers.

Go-live criteria: All "Critical" tests pass. No "High" tests fail. At least 1 full end-to-end test (lead → booking → SMS) completed successfully.
14
Share the client portal link with Rudy

Send Rudy a link to client-portal-bighatlawn.html (hosted on CDN) so he can start seeing his dashboard.

https://cdn.jsdelivr.net/gh/kalelra/512ai@main/files/client-portal-bighatlawn.html
